# Doc-31: 課金・サブスク設計

## 1. サブスクリプションモデル
月額の自動継続課金を基本モデルとする。決済プロバイダとしてStripeを利用する。

## 2. 課金タイミング
- 新規契約時：即時請求（またはトライアル期間経過後の自動請求開始）。
- 次回更新時：初回契約日から起算して1ヶ月ごとの自動継続課金。

## 3. プラン定義 (`Plan` モデル)
| 項目 | 型 | 説明 |
| :--- | :--- | :--- |
| `name` | String | プラン名（Standard, Premium等） |
| `price` | Integer | 月額料金 |
| `maxStores` | Integer | 登録可能な最大店舗数 |
| `maxUsers` | Integer | 登録可能な最大ユーザー（スタッフ・キャスト）数 |

## 4. データ保持ポリシーとアーキテクチャ
- **Tenant（テナント）**: NightOpsを利用する顧客ごとのハブとなるエンティティ。ここに現在の `planId`, `status` を保持する。
- **Subscription（サブスクリプション）**: Stripeと直接結びつく請求レコード。一つのテナントに対して原則一つアクティブなものが存在する。
- **BillingHistory（課金履歴）**: 成功・失敗問わず、毎月の処理ごとの監査ログ。
