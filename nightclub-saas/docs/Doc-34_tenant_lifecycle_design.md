# Doc-34: テナントライフサイクル設計

## 1. 状態遷移フロー
各テナント (`Tenant.status`) は以下の5つの状態を持つものとする。

1. **trial** : サインアップ直後の無料トライアル期間。
2. **active** : クレジットカード決済が成功し、正常に稼働している状態。
3. **past_due** : クレジットカード引き落としに失敗しているが、猶予期間（リトライ期間）中。
4. **suspended** : 猶予期間が過ぎ、サービス利用を強制停止された状態。
5. **canceled** : 解約済み。またはSystemAdminによるBAN。

## 2. アクセス制御（ミドルウェア層）
- 各テナントのAPI利用リクエストは、Global Middlewareまたは専用Guard (`TenantStatusGuard`) によってインターセプトされる。
- **active** : フルアクセス可能。
- **trial** : 試験用の設定・制限に基づきアクセス可能。（本番では期限監視バッチ等を追加予定）
- **past_due** : 利用は可能だが、システム上で次回決済を促す警告フラグをフロントエンドに返却（または一部更新制限）。
- **suspended** : すべての書き込み・設定API (`POST`, `PUT`, `DELETE`) の実行を禁止し、`403 Forbidden` を返す。管理画面の表示など最小限の参照（データの持ち出し等）は可能。
- **canceled** : ログイン後に自テナントデータへの参照のみ許可、新規入力や業務処理は不可。
