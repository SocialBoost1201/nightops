# Doc-30: SaaS基盤要件定義

## 1. 目的
NightOpsを単一テナント想定のシステムから、「販売可能なサブスクリプション型SaaS」へと進化させる。

## 2. 絶対条件
- 既存のM1/M2等の設計を破壊しないこと。
- 現状の `tenantId` 依存のデータモデリングを中核として維持する。
- 決済基盤は Stripe を前提とし、自前でのクレジットカード情報保持や決済システム構築は行わない。
- 課金ロジック・Webhookイベント処理は必ず冪等性を担保すること。
- 後からのスケールアップ・ダウン（プラン変更など）に耐えうるアーキテクチャとする。

## 3. 主要要件
### 3.1 契約（テナント）管理機能
- アカウント作成時、各々に論理的に独立した「テナント（契約単位）」を割り当てる。
- 全テナント横断のSystemAdminが、各テナントの契約ステータスを監視・制御できるようにする。

### 3.2 課金・サブスクリプション
- 月額課金（プロラタ、段階定額等）をサポート可能にする。
- Stripe へのリダイレクト(Checkout) または埋め込みコンポーネントを用いた安全な決済。
- 支払いの成功/失敗などの状態をリアルタイムで同期する仕組み(Webhook)。

### 3.3 プラン別機能制御
- 定義済みプラン（例：Basic, Premiumなど）に基づく機能制限の適用。
- 登録可能ユーザー数（Cast / Staff）や登録可能店舗数等の上限制御。

### 3.4 ライフサイクルと利用制限
- `trial` → `active` → `past_due` → `suspended` → `canceled` の状態遷移を厳格に管理する。
