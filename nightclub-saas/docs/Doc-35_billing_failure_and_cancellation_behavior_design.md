# Doc-35: 課金失敗・解約時挙動設計

## 1. 課金失敗 (payment_failed) の挙動
Stripe Webhook で `invoice.payment_failed` を受け取った場合：
- テナントの状態をただちに `past_due` へ遷移する。
- ユーザーに猶予期間（通常数日間〜1週間程度、Stripe設定に依存）があることを通知。
- Stripeのロジックで再請求が成功（`payment_succeeded`）した場合は、再び `active` に戻る。

## 2. 強制停止 (suspended) への移行
Stripeのリトライスパンがすべて終了し、サブスクリプションが自動停止となった際：
- テナントを強制的に `suspended` に遷移する。
- この状態になると、打刻や伝票入力などの日々の業務処理系APIが全て書き込みエラーとなる。ユーザーは新しいクレジットカード情報を登録して復活させる必要がある。

## 3. 解約時 (canceled) のデータ保持ポリシー
- 契約解除後、テナント情報は `canceled` として存続する（物理削除はしない）。
- ただし、法定保持期間（日本の税法や労基法上の3年〜7年）を考慮し、退会後一定期間は参照できるようにする。
- 再度サービスを利用する場合は、基本的には新規テナント発行か、あるいは特別対応で復活処理を行う（システム管理画面から `activate` 可能）。
