# Doc-33: プラン・機能制御設計

## 1. 目的
NightOps内での全リソース（店舗、キャスト、スタッフ等）への制限を、選択したプランに基づいて制御できるようにする。テナントが現在利用しているプランに応じて、API側でリソース作成を制限する。

## 2. 制御対象ロジック
- 最大構成数（`maxStores`, `maxUsers` etc.）の制御：
  - 例外発生処理を追加し、作成時にプランの上限を超える場合は `ConflictException` ('PLAN_LIMIT_EXCEEDED') をスローする。
- プレミアム機能の制限（`FeatureKey`）：
  - `PlanFeature` に定義された Key が `enabled=true` の場合のみ特定モジュール（高度な集計分析など）へアクセス可能。

## 3. 実装方針
- リソース作成を行う各種Service（`UserService`, `StoreService` 等）にて、常に利用中のプランの上限値を取得・チェックする共通メソッドを噛ませる。
- 将来的な拡張性のため、`tenantId` をキーにキャッシュ（Redis等）へ保持しておく設計を検討（初期はDB直接参照でも可）。
