## 1. 背景と目的

### 1-1. 背景
現状は紙の出勤予定表とタイムカード、売上はExcel入力。データが分断し集計と給与計算に手間とミスが発生する。

### 1-2. 目的
シフト提出、勤怠確定、売上伝票入力、顧客台帳、ランキング、給与計算を一元化し、日次締めと月次確定で改ざんとミスを防ぐ。

### 1-3. 成功条件
1. 2週間シフト提出と承認が現場で回る
2. キャストは出勤のみ、スタッフは出退勤で運用できる
3. 伝票入力から小計、税サービス、総計が自動計算され、締め後ロックされる
4. 小計ベースのランキングと給与集計が自動で出る
5. 修正は申請承認と監査ログで統制される
6. 店舗ごとの差分（料金、丸め、バック）を設定で吸収できる

---

## 2. スコープ

### 2-1. MVPで提供する機能
1. マルチテナント基盤（tenantId分離、RBAC、監査ログ）
2. アカウント管理（管理者発行ID、初回PW変更、無効化、再発行）
3. シフト提出（2週間）と承認（店長か社長の片方で確定）
4. 勤怠（キャスト出勤のみ、スタッフ出退勤、キャストあがり時間は管理者確定）
5. 日次締め（勤怠、売上、締め後ロック、修正申請承認）
6. 売上伝票入力（アニモ料金テンプレ、自由明細、顧客名連動）
7. 計算（小計、税サービス＝小計×1.32、総計＝1000円単位四捨五入）
8. 顧客台帳（名前、メモ、来店履歴、重複統合）
9. ランキングと集計（日次、月次、CSV出力）
10. 給与集計（月次、Type1時給＋バック、Type2歩合＋バック、個別バック率、CSV出力、月次確定ロック）
11. 通知（提出期限、未承認、未締め、未打刻、未あがり、未退勤）

### 2-2. Phase2で拡張する機能
1. 複数本指名時の売上按分（均等割、比率指定）
2. ドリンクを明細化し自動杯数算出（ボトル換算など）
3. 管理者スマホアプリの強化（売上入力補助、承認、締め）
4. 課金（サブスク）、契約管理、請求書、支払い連携
5. 給与明細PDF、振込データ出力
6. Windowsネイティブ管理アプリ（必要なら）

## 4. マルチテナント要件

### 4-1. データ分離
全テーブルに `tenantId` を持ち、APIは必ず `tenantId` スコープを強制する。

### 4-2. 店舗設定（StoreSettings）

| 設定項目 | 説明 | MVP対応 |
|---|---|---|
| storeCode | ログインIDの接頭辞 | ○ |
| 締め時刻 | 運用目安 | ○ |
| 料金マスタ | 項目/単価/単位 | ○ |
| 税サービス係数 | デフォルト1.32 | ○ |
| 総計丸めルール | 1000円単位四捨五入 | ○ |
| バック標準単価 | 場内、ドリンク | ○ |
| シフト周期 | MVPは2週間固定 | △（将来設定化） |
| 通知ポリシー | 提出期限など | ○ |

## 5. アカウント要件

### 5-1. ログインID形式
```
{storeCode}-{4桁連番}
例: anim-0001
```
- 欠番を埋めない
- ID再利用しない

### 5-2. パスワード
- 初期PWは自動生成、配布用に一度だけ表示
- 初回ログインで変更必須
- 管理者による再発行を提供

### 5-3. 退店処理
- Accountを `INACTIVE` にしてログイン不可
- 過去データは保持（削除しない）

### 5-4. セキュリティ
- パスワードハッシュ（Argon2 または bcrypt）
- ログイン試行回数制限
- セッション管理（JWT＋リフレッシュトークン）

---

## 6. 勤怠要件

### 6-1. シフト提出
- 2週間単位の提出
- 日付ごとに予定出勤時刻、予定退勤時刻、メモを任意入力
- 変更は変更申請（理由必須）

### 6-2. 承認
- 店長か社長のいずれかが承認すれば確定
- 確定後ロック

### 6-3. 打刻
- キャスト: 出勤ボタンのみ（長押し）
- スタッフ: 出勤/退勤ボタン（長押し）
- 位置情報、QR不要

### 6-4. あがり時間
- キャストは退勤ボタンなし
- 管理者が当日一覧であがり時間を入力し確定

### 6-5. 日次締め（勤怠）
- 営業日ごとに締め
- 締め前チェック: キャストあがり未入力、スタッフ退勤未入力、予定なし出勤
- 締め後は編集不可
- 修正は申請承認で行う

---

## 7. 売上要件

### 7-1. 入力運用
- 営業終了後、店長がWeb管理画面で伝票入力
- 営業日ごとに締め
- 締め後ロック、修正は申請承認

### 7-2. 伝票入力項目

| 項目 | 必須 | 説明 |
|---|---|---|
| 営業日 | ○ | |
| 卓番 | - | 任意 |
| 顧客名 | 推奨 | 顧客台帳に紐付き |
| 人数 | ○ | |
| 本指名キャスト（主担当） | ○ | 売上帰属 |
| 明細行 | ○ | 項目、数量、単価、金額（自動） |
| 自由明細 | - | 名称、数量、単価 |
| ドリンク杯数 | - | キャスト別に杯数を別入力（MVP） |

### 7-3. アニモ料金テンプレート

| 項目 | 単価(円) | 単位 |
|---|---|---|
| セット（ハウス） | 7,000 | 1名60分 |
| セット（フリー）A | 4,000 | 1名60分 |
| セット（フリー）B | 5,000 | 1名60分 |
| T・C | 6,000 | 1名60分 |
| 本指名 | 3,000 | 1本 |
| 同伴 | 5,000 | 1本 |
| 場内指名 | 2,000 | 1本 |
| P1 | 3,000 | 1回 |
| P2 | 3,500 | 1回 |
| SP | 2,000 | 1回 |
| SP2 | 2,500 | 1回 |
| SP3 | 3,000 | 1回 |
| SP4 | 3,500 | 1回 |
| T・S 60（延長） | 7,000 | 1名60分 |
| T・S 30（延長） | 3,500 | 1名30分 |
| 自由明細 | 任意 | ボトル等 |

### 7-4. 計算ルール

```
小計 ＝ 明細合計
税サービス ＝ 小計 × 1.32
総計 ＝ 1000円単位四捨五入（端数500以上繰上げ、499以下繰下げ）
```

> 給与とランキングの売上対象は **小計のみ**（税サービス、総計丸めは対象外）

### 7-5. 売上帰属（MVP仮定）
- 小計は主担当（本指名キャスト）へ100%帰属
- 按分はPhase2

---

## 8. 顧客要件

- 顧客名をキーに台帳化
- 候補表示（オートサジェスト）と検索を提供
- 顧客詳細: 来店履歴（伝票一覧）、累計小計、メモ
- 重複統合は管理者のみ実行、統合履歴を保持

---

## 9. ランキングと集計

### 9-1. 日次
- キャスト別小計
- 指名本数（本指名/同伴/場内）
- ドリンク杯数
- 勤怠の未入力アラート
- 未締めアラート

### 9-2. 月次
- キャスト別小計累計
- 指名本数累計
- 出勤日数
- 勤務時間
- CSV出力

### 9-3. ランキング
- 期間指定で小計順
- 指名列、出勤列、ドリンク列を表示

## 10. 給与要件（個人別混在）

### 10-1. 給与タイプ

| タイプ | 計算方式 |
|---|---|
| Type1 | 時給 ＋ バック |
| Type2 | 完全歩合 ＋ バック |

### 10-2. 管理者が登録する個人条件（適用期間付き）

| 項目 | 説明 | デフォルト |
|---|---|---|
| 給与タイプ | Type1 / Type2 | — |
| 時給 | Type1のみ | — |
| 売上バック率 | 個別 | — |
| 場内バック単価 | 個人上書き可 | 1,000円 |
| ドリンクバック単価 | 個人上書き可 | 100円 |
| 適用開始日 | 必須 | — |
| 適用終了日 | 任意 | — |

> スタッフも個人別時給を設定可能

### 10-3. 計算ロジック

```
【対象データ】日次締め済みの勤怠と売上のみ

【勤務時間】
  キャスト: 出勤打刻 〜 あがり確定
  スタッフ: 出勤打刻 〜 退勤打刻

【売上対象】主担当に帰属した小計

【バック計算】
  売上バック ＝ 小計 × 個別率
  場内バック ＝ 回数 × 単価
  ドリンクバック ＝ 杯数 × 単価

【総支給】
  Type1 ＝ 時給分 ＋ 売上バック ＋ 場内バック ＋ ドリンクバック ＋ 手当 − 控除
  Type2 ＝ 売上バック ＋ 場内バック ＋ ドリンクバック ＋ 手当 − 控除
```

### 10-4. 月次確定
- 月次確定でロック
- 再確定は監査ログに記録

## 11. 通知要件（MVP）

| 通知種別 | 対象 | タイミング |
|---|---|---|
| シフト提出期限 | Cast, Staff | 期限の24時間前 |
| 未承認シフト | Manager | 提出から24時間未承認 |
| 未締め営業日 | Manager | 翌日の所定時刻 |
| 未打刻 | Cast, Staff | シフト予定あり＋出勤打刻なし |
| 未あがり | Manager | キャスト出勤済み＋あがり未入力 |
| 未退勤 | Manager | スタッフ出勤済み＋退勤未入力 |

> MVPではアプリ内通知のみ。Phase2でFCMプッシュ通知を追加。

---

## 12. 修正申請と監査

### 12-1. 修正対象
- シフト、打刻、あがり、伝票、顧客統合、給与条件、月次確定

### 12-2. 申請要件
- 理由必須
- 承認: 店長か社長（給与条件と月次確定はAdmin以上）

### 12-3. 監査ログ
- 操作者、日時、対象、変更前後、理由
- 削除不可
- テナント分離

---

## 13. 非機能要件（販売品質）

### 13-1. セキュリティ
- テナント分離、RBAC、最小権限、ハッシュ、操作監査

### 13-2. 信頼性
- 自動バックアップ（日次）
- 障害時復旧手順

### 13-3. 観測性
- 例外監視、運用ログ、アラート

### 13-4. エクスポート
- 売上、勤怠、給与のCSV出力

### 13-5. 個人情報
- MVPは本名、住所、口座、マイナンバーを保持しない

## 16. 用語集

| 用語 | 定義 |
|---|---|
| テナント | 本SaaSを利用する1店舗（契約単位） |
| キャスト | 接客を行うスタッフ（Cast） |
| スタッフ | 接客以外の業務を行う従業員（Staff） |
| あがり | キャストの退勤（業務終了）時刻 |
| 本指名 | 顧客が特定キャストを指名して来店 |
| 同伴 | キャストと顧客が店外から同伴して来店 |
| 場内指名 | 店内で顧客がキャストを指名 |
| バック | 指名数や売上に応じてキャストに還元される報酬 |
| 日次締め | 営業日単位でデータを確定しロックする処理 |
| 月次確定 | 月単位で給与データを確定しロックする処理 |
| 小計 | 伝票の明細合計（税・サービス計算前） |
